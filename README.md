# داشبورد پایش عملیات «نصر ۲» و «صاعقه»

داشبورد تعاملی برای پایش رسانه‌ای بیانیه‌های رسمی نظامی جمهوری اسلامی ایران — سپاه پاسداران (عملیات «نصر ۲») و ارتش جمهوری اسلامی (عملیات «صاعقه») — با نمایش مکان، تاریخ و جزئیات هر حملهٔ ادعایی روی نقشه.

🔗 **سایت زنده:** [nasrdashboard.com](https://nasrdashboard.com)

> ⚠️ **مهم:** تمام داده‌های این داشبورد بازتاب مستقیم ادعاهای منتشرشده توسط منابع رسمی نظامی (روابط عمومی سپاه پاسداران و ارتش) است. این داده‌ها **راستی‌آزمایی مستقل ندارند** و صرفاً برای مقاصد پایش رسانه‌ای گردآوری شده‌اند.

---

## ویژگی‌ها

- نقشهٔ تعاملی (Leaflet) با نشانگر مجزا برای هر منبع — دایره برای سپاه، لوزی برای ارتش
- تایم‌لاین با نمایش تجمعی یا تک‌روزه و پخش خودکار
- فیلتر بر اساس کشور هدف، نیروی عامل (زمینی/دریایی/هوافضا)، و منبع بیانیه
- ۴ نسخهٔ زبان/دستگاه: دسکتاپ فارسی، موبایل فارسی، موبایل انگلیسی، موبایل عربی
- تشخیص خودکار دستگاه در صفحهٔ فرود و هدایت به نسخهٔ مناسب
- به‌روزرسانی خودکار هر ۶ ساعت از فیدهای RSS رسمی، با استخراج ساختاریافتهٔ داده توسط هوش مصنوعی

## ساختار پروژه

```
/index.html                 صفحهٔ فرود + تشخیص خودکار دستگاه
/index-fa-desktop.html      داشبورد کامل، فارسی، دسکتاپ
/index-fa-mobile.html       داشبورد موبایل، فارسی
/index-en-mobile.html       داشبورد موبایل، انگلیسی
/index-ar-mobile.html       داشبورد موبایل، عربی

/data/
  locations.json            گازتیر مکان‌ها (نام، کشور، مختصات)
  countries.json            برچسب‌های چندزبانهٔ کشورها
  sources.json               برچسب‌های منبع (سپاه/ارتش)
  forces.json                برچسب‌های نوع نیرو
  events.json                کل رویدادها (منبع اصلی داده‌های نمایش‌داده‌شده)
  meta.json                  بازهٔ تاریخ و زمان آخرین به‌روزرسانی
  seen-guids.json            آرشیو شناسه‌های پردازش‌شده (برای جلوگیری از تکرار)
  pending-review.json        رویدادهای نیازمند بررسی دستی

/scripts/
  sync-rss.mjs               اسکریپت اصلی پایپ‌لاین همگام‌سازی
  sync-rss.test.mjs          تست‌های خودکار پایپ‌لاین
  README.md                  مستندات فنی کامل پایپ‌لاین و محدودیت‌های شناخته‌شده

/.github/workflows/
  sync-sepahnews.yml         اجرای خودکار پایپ‌لاین (هر ۶ ساعت + دستی)
```

## معماری پایپ‌لاین همگام‌سازی

هر ۶ ساعت، یک GitHub Action:

1. فید RSS هر دو منبع (سپاه‌نیوز و آجا) را می‌خواند
2. آیتم‌های تکراری یا نامرتبط را فیلتر می‌کند
3. متن کامل هر خبر مرتبط را واکشی می‌کند
4. با استفاده از یک مدل زبانی (از طریق [OpenRouter](https://openrouter.ir)) داده را به فرمت ساختاریافته (مکان، نیرو، سلاح، نتیجه و…) تبدیل می‌کند
5. خروجی معتبر را به `data/events.json` اضافه می‌کند؛ موارد نامطمئن به `data/pending-review.json` می‌روند و یک [Issue](../../issues?q=is%3Aissue+label%3Aneeds-review) برای بررسی دستی باز می‌شود

جزئیات کامل فنی، از جمله محدودیت‌های شناخته‌شده و نحوهٔ اجرای دستی، در [`scripts/README.md`](scripts/README.md) مستند شده.

## توسعهٔ محلی

هر ۴ فایل HTML مستقیماً از فایل‌های `data/*.json` می‌خوانند — کافی است فایل‌ها را روی یک سرور استاتیک محلی سرو کنید:

```bash
npx serve .
```

برای اجرای پایپ‌لاین همگام‌سازی به‌صورت محلی، به `scripts/README.md` مراجعه کنید (نیازمند تنظیم متغیرهای محیطی برای کلید OpenRouter و در صورت نیاز به منبع ارتش، دسترسی به پروکسی مربوطه).

## مشارکت

اگر در بررسی موارد `pending-review.json` یا رفع مشکلات پایپ‌لاین (لیبل [`needs-review`](../../issues?q=is%3Aissue+label%3Aneeds-review)) تمایل به کمک دارید، خوش‌آمدید.
